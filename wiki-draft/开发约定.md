# 开发约定

语言：[简体中文](./首页) · [English](./Home_EN) · [日本語](./ホーム)

## 代码风格

- 代码注释和 GUI 文案以中文为主
- `README.md` 是权威来源；`README_EN.md` 和 `README_JP.md` 是维护的翻译
- 默认使用 ASCII，仅在文件本身已是该字符集时引入非 ASCII 或其它 Unicode 字符
- 只在代码不自明时加简短注释，避免「把值赋给变量」之类的空叙述；复杂块前可留一行简短定向注释

## 关键返回值约定

- **模板动作返回值是 `2`/`1`，不是布尔值。** `2` 表示成功/找到/点击；`1` 表示未找到/继续尝试/取消。`find_win()` 是例外：返回几何元组或 `None`
- **模板文件名用 `picture` 参数，不是 `name`。** `click_item_with_result(self, picture, name)` 和 `find_item_with_result(...)` 从 `<picture>_1.png` 开始连续发现，直到第一个缺失的编号。`name` 只是日志标签。编号必须从 `_1` 开始且连续
- 同一模板组的全部变体必须与同一帧截图比较，只执行整组中匹配分数全局最高的候选；不要逐模板重新截图或按发现顺序提前返回
- 截图与模板应用相同的 3x3 `GaussianBlur` 后使用 `TM_SQDIFF_NORMED`；匹配分数必须严格 `> 0.8`，等于 `0.8` 不算匹配

## 运行/停止状态

- 运行/停止状态是 per worker 的。`_running()` 是 `_active and not _stop_event.is_set()`
- 使用 `worker.stop()`；不要重新引入全局 `guaji` 标志
- `start()` 在旧线程仍在运行时拒绝重启

## LP 恢复值

- LP 恢复值是 drinks + 1。GUI 对 Link Raid 显示 0-10，对晶花显示 0-8，但传递 `shown + 1`
- 内部 `1` 表示零喝药并在第一次耗尽时停止

## 日志

- 运行时调试输出用 `logging.getLogger(__name__)`，不要用 `print()`
- GUI 日志处理器支持运行时选择 DEBUG/INFO/WARNING/ERROR/CRITICAL 等级；文件处理器始终以 DEBUG 级别写入 `logs/`
- 有控制台时控制台默认 WARNING，`MAGIA_LOG_LEVEL` 环境变量可覆盖；使用 `--windowed` 构建时没有控制台，缺少 stderr 必须安全跳过，不能影响 GUI 启动
- worker 面向用户的运行状态消息仍走 `signal.emit`（GUI 日志框），不受 GUI/控制台 logging 等级过滤，也不要用 `logging` 取代
- 各文件 `if __name__ == "__main__"` 诊断块可保留 `print`，那是 CLI 直读输出，不属于运行时噪音

## GUI 参数

- GUI 参数由注册表驱动，没有模块级参数全局变量
- `ParamSpec` 的 `kind` 支持 `choice`、`lp_recover`、`int`
- `required_templates` 路径可使用 `{level_choice}` 这类占位符，但占位符名称必须对应同一 worker 中已声明的 `ParamSpec.key`
- 新增控件类型只需在 `registry.py` 加一个 `kind`，并在 `main.py` 的 `_build_param_widget` 加对应分支

## 新增挂机模式

1. 写一个 `BaseWorker` 子类文件，放在 `src/workers/` 下
2. 用 `@register` 装饰器声明显示名、起始界面提示、参数列表和必需模板
3. 在 `src/workers/__init__.py` 中加一行 import
4. `main.py` 的 GUI 会自动出现对应按钮和参数控件，无需改 GUI 代码

worker 必须无参数可构造，且必须通过 `_run_safely()` 路由 `run()`。

## 用户输入保护与等待恢复

- 用户键盘输入或累计较大的鼠标移动会暂停自动化，连续空闲 5 秒后才恢复
- 所有新的 PyAutoGUI 输入路径都必须先调用 `_wait_for_user_idle()`，在 `_automation_input()` 上下文中执行自动化鼠标操作，并在成功后记录最后一次自动化位置
- 在同一次点击模板等待中，每连续未匹配 5 秒都要观察游戏客户区 2 秒。若变化像素超过 50%，视为战斗动画，仅跳过本轮恢复；否则在最后一个安全的自动化位置执行一次恢复点击
- 上述 5 秒周期必须持续循环，直到模板成功、等待超时或任务取消，绝不能只运行一次恢复检查

## DPI 敏感的导入顺序

保持 `src.workers` 懒导入。启动必须保持 `QApplication(...) -> mywindow() -> get_worker_registry()`。更早导入 workers 会导入 PyAutoGUI，可能阻止 Qt 设置 Windows DPI awareness。

## 模板包规则

- `aim/` 是运行时 Windows 目录联接，不是真实模板文件夹
- 不要手动合并/重命名 `aim/` 或把派生模板当作真源编辑。新增变体应加到 2K 源 pack 再重新生成
- 不要删除空 pack 的 `.gitkeep` 占位。缺失的 pack 目录不会被缩放器重建
- 新增语言时在 `language_switcher.LANG_LABELS` 加一项，GUI 会自动读取

## 游戏标题

游戏窗口标题硬编码为 `MadokaExedra`。更改它需要同步更新 `main.py`、`click_action.py` 和 `click_behavior.py`，或先集中化它。

## Git 与本地产物

- `.gitignore` 忽略 `aim/`、`language/active.json`、`__pycache__/`、`*.pyc` 和 `logs/`
- 派生 PNG、`.source_hashes.json`、`build/`、`dist/`、生成的 `main.spec` 可能出现为非跟踪文件，绝不误暂存
- `tools/ImageMagick/` 已入库且为发行包所必需
- 仓库有 `main`、`beta` 等分支。绝不单独从当前分支推断目标分支或发布通道
- commit 消息是简明中文，含简短主题和正文

## 检查清单

本项目没有正式测试套件、lint/typecheck 配置或 CI。改动后请手动执行适用的非游戏检查：

- Python 编译/导入检查
- worker 注册表与模板校验
- 更新 ZIP/解压校验
- 锁/更新器检查
- AMD64 PE 校验
- 打包后冒烟启动
