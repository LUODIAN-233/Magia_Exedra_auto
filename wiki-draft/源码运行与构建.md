# 源码运行与构建

语言：[简体中文](./首页) · [English](./Home_EN) · [日本語](./ホーム)

## 运行环境

- 操作系统：Windows x86-64 / AMD64
- Python：3.x（建议 3.10+）

## 依赖安装

运行时依赖声明在 `requirements.txt`：

```bash
pip install -r requirements.txt
```

构建发行包还需 `pyinstaller`（已在 `requirements.txt` 中注释掉，按需取消注释或单独安装）：

```bash
pip install pyinstaller
```

素材缩放使用仓库内提交的 `tools/ImageMagick/magick.exe`，无需额外安装；若系统 PATH 中已有 `magick` 也可使用。

## 从源码运行

```bash
python main.py
```

`main.py` 是入口，启动时会将当前工作目录切换到脚本所在目录，保证相对运行时路径（`./aim/...`、`./resource/...`）能正确解析。启动过程中会调用 `language_switcher.ensure_active()`，恢复或创建 `aim/` 目录联接。

启动时还会调用 `log_setup.configure_logging()` 配置日志：控制台默认只输出 WARNING 及以上（避免识图重试刷屏），DEBUG 级别滚动写入 `logs/` 目录便于事后排查；设置环境变量 `MAGIA_LOG_LEVEL=DEBUG` 可让控制台输出全部日志。

## 打包构建

使用 PyInstaller 的 onedir 模式打包：

```bash
pyinstaller -D -i resource/main.ico -n Magia_Exedra_auto main.py
```

`-n Magia_Exedra_auto` 把构建出的入口 exe 命名为项目名 `Magia_Exedra_auto.exe`（而非默认的 `main.exe`），这是发行包里实际使用的入口可执行文件。此命令只生成 PyInstaller 的 onedir 输出（`dist/Magia_Exedra_auto/` 下含 `Magia_Exedra_auto.exe` 和 `_internal/`）。可分发的完整包还需把版本控制的运行时文件放在 `Magia_Exedra_auto.exe` 同级：

- `resource/`（图标等资源）
- `language/`（模板包，含 `.gitkeep` 占位）
- `tools/`（ImageMagick 可执行文件）

`Magia_Exedra_auto.exe` 必须位于 ZIP 根目录。不要把 `aim/`、`active.json`、`.source_hashes.json`、本地生成的派生模板、缓存或 Git 构建元数据打入发行包。

## 打包环境注意事项

- 在 frozen 模式下，必须在导入 `cv2` 前设置环境变量 `OPENCV_SKIP_PYTHON_LOADER=1`。`main.py` 已在入口处理此事，请勿打乱导入顺序。
- **DPI 敏感的导入顺序**：保持 `src.workers` 懒导入。启动流程必须是 `QApplication(...) -> mywindow() -> get_worker_registry()`。更早导入 workers 会触发 PyAutoGUI 导入，可能导致 Qt 无法正确设置 Windows DPI awareness。`main.py` 在导入期、`QApplication` 之前调用 `log_setup.configure_logging()`，该模块只依赖标准库，安全且必须先于业务模块导入，以便捕获所有 logger。

## 验证检查

本项目没有正式测试套件、lint 或 CI 配置。改动后请手动执行适用的非游戏检查：

- Python 编译与导入检查
- worker 注册表与模板校验
- 更新 ZIP 与解压校验
- 锁与更新器检查
- AMD64 PE 校验
- 打包后冒烟启动
