# 開発規約

言語：[简体中文](./首页) · [English](./Home_EN) · [日本語](./ホーム)

## コードスタイル

- コードコメントと GUI 文章は中国語優先です
- `README.md` が権威あるソースです。`README_EN.md` と `README_JP.md` は維持された翻訳です
- デフォルトは ASCII とし、明確な理由がありファイルが既にその文字セットである場合のみ非 ASCII やその他の Unicode を導入します
- コードが自明でない場合のみ簡潔なコメントを追加し、「変数に値を代入する」ような空の叙述を避けます

## 主要な戻り値の規約

- **テンプレートアクションの戻り値は `2`/`1` であり、ブール値ではありません。** `2` は成功/発見/クリック済み、`1` は未発見/再試行継続/キャンセルを意味します。`find_win()` が例外で、ジオメトリタプルまたは `None` を返します
- **テンプレートファイル名は `name` ではなく `picture` 引数を使用します。** `click_item_with_result(self, picture, name)` と `find_item_with_result(...)` は `<picture>_1.png`、`<picture>_2.png` と順番に試し、最初に見つからない番号で停止します。`name` はログラベルのみです。番号は `_1` から開始し連続している必要があります
- マッチング閾値は `click_behavior.routine()` と `routine_only_find()` で厳密に `> 0.8` です

## 実行/停止状態

- 実行/停止状態は worker ごとです。`_running()` は `_active and not _stop_event.is_set()` です
- `worker.stop()` を使用してください。グローバルな `guaji` フラグを再導入しないでください
- `start()` は旧スレッドがまだ実行中の場合は再起動を拒否します

## LP 回復値

- LP 回復値は drinks + 1 です。GUI は Link Raid で 0-10、Crystalis で 0-8 を表示しますが、`shown + 1` を渡します
- 内部 `1` は薬を使用しないことを意味し、最初の枯渇時に停止します

## ログ

- 実行時のデバッグ出力には `print()` ではなく `logging.getLogger(__name__)` を使用します
- コンソールはデフォルト WARNING、ファイルは `logs/` に DEBUG レベルで出力します。環境変数 `MAGIA_LOG_LEVEL` でコンソールレベルを上書きできます
- worker のユーザー向け状態メッセージは引き続き `signal.emit`（GUI ロログ欄）で行い、`logging` で置き換えないでください
- 各ファイルの `if __name__ == "__main__"` 診断ブロックは `print` を維持して構いません。これは CLI 直読出力であり、実行時ノイズではありません

## GUI パラメータ

- GUI パラメータはレジストリ駆動です。モジュールレベルのパラメータグローバルはありません
- `ParamSpec.kind` は `choice`、`lp_recover`、`int` をサポートします
- 新しいコントロールタイプを追加するには、`registry.py` に `kind` を追加し、`main.py` の `_build_param_widget` に対応する分岐を追加します

## 新しい周回モードの追加

1. `src/workers/` 配下に `BaseWorker` サブクラスファイルを作成します
2. `@register` でデコレートし、表示名、開始画面のヒント、パラメータリスト、必須テンプレートを宣言します
3. `src/workers/__init__.py` にインポート行を1行追加します
4. `main.py` の GUI に対応するボタンとパラメータコントロールが自動的に表示されます。GUI コードの変更は不要です

worker は引数なしで構築可能であり、`run()` を `_run_safely()` 経由でルーティングする必要があります。

## DPI に敏感なインポート順序

`src.workers` は遅延インポートのままにしてください。起動は `QApplication(...) -> mywindow() -> get_worker_registry()` の順序を維持する必要があります。workers を先にインポートすると PyAutoGUI が読み込まれ、Qt が Windows DPI awareness を設定できなくなる可能性があります。

## テンプレート pack のルール

- `aim/` は実行時の Windows ディレクトリジャンクションであり、実テンプレートフォルダではありません
- `aim/` を手動でマージ/リネームしたり、派生テンプレートを真のソースとして編集しないでください。新規バリアントは 2K ソース pack に追加して再生成してください
- 空 pack の `.gitkeep` プレースホルダを削除しないでください。欠落した pack ディレクトリはスケーラによって再作成されません
- 言語を追加する際は `language_switcher.LANG_LABELS` にエントリを追加してください。GUI が自動的に読み取ります

## ゲームタイトル

ゲームウィンドウのタイトルは `MadokaExedra` としてハードコードされています。変更する場合は `main.py`、`click_action.py`、`click_behavior.py` を更新するか、まず一元化する必要があります。

## Git とローカル産物

- `.gitignore` は `aim/`、`language/active.json`、`__pycache__/`、`*.pyc`、`logs/` を無視します
- 派生 PNG、`.source_hashes.json`、`build/`、`dist/`、生成された `main.spec` は非追跡ファイルとして現れることがあります。誤ってステージしないでください
- `tools/ImageMagick/` はコミットされており、リリースパッケージに必要です
- リポジトリには `main`、`beta` などのブランチがあります。現在のブランチのみから対象ブランチやリリースチャネルを推測しないでください
- コミットメッセージは簡潔な主題と本文を持つ平易な中国語です

## チェックリスト

正式なテストスイート、lint/typecheck 設定、CI はありません。変更後は該当する非ゲームチェックを手動で実行してください：

- Python コンパイル/インポートチェック
- worker レジストリとテンプレート検証
- 更新 ZIP/展開チェック
- ロック/アップデータチェック
- AMD64 PE 検証
- パッケージ化後のスモークスタート
