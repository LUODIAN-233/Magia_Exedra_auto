# ソース実行とビルド

言語：[简体中文](./首页) · [English](./Home_EN) · [日本語](./ホーム)

## 実行環境

- OS：Windows x86-64 / AMD64
- Python：3.x（3.10 以上を推奨）

## 依存関係のインストール

本プロジェクトには `requirements.txt` がありません。以下の実行時依存関係を手動でインストールしてください：

```bash
pip install pyautogui PySide6 opencv-python numpy pywinctl Pillow
```

リリース版のビルドには以下も必要です：

```bash
pip install pyinstaller
```

素材のスケーリングはリポジトリ内にコミットされた `tools/ImageMagick/magick.exe` を使用し、別途インストールは不要です。PATH 上の `magick` も利用可能です。

## ソースからの実行

```bash
python main.py
```

`main.py` がエントリポイントです。起動時にカレントディレクトリをスクリプト配置ディレクトリに切り替え、相対実行時パス（`./aim/...`、`./resource/...`）が正しく解決されるようにします。起動時に `language_switcher.ensure_active()` が呼ばれ、`aim/` ディレクトリジャンクションを復元または作成します。

## パッケージ化

PyInstaller の onedir モードを使用します：

```bash
pyinstaller -D -i resource/main.ico main.py
```

このコマンドは PyInstaller の onedir 出力（`dist/main/` に `main.exe` と `_internal/` を含む）のみを生成します。配布可能なパッケージには、`main.exe` と同階層にバージョン管理された実行時ファイルも配置する必要があります：

- `resource/`（アイコン等のリソース）
- `language/`（テンプレート pack、`.gitkeep` プレースホルダを含む）
- `tools/`（ImageMagick 実行ファイル）

`main.exe` は ZIP のルートに配置してください。`aim/`、`active.json`、`.source_hashes.json`、ローカル生成の派生テンプレート、キャッシュ、Git/ビルドメタデータはリリースパッケージに含めないでください。

## パッケージ化の注意事項

- frozen モードでは、`cv2` がインポートされる前に環境変数 `OPENCV_SKIP_PYTHON_LOADER=1` を設定する必要があります。`main.py` がエントリポイントで処理していますが、インポート順序を変えないでください。
- **DPI に敏感なインポート順序**：`src.workers` は遅延インポートのままにしてください。起動は `QApplication(...) -> mywindow() -> get_worker_registry()` の順序を維持する必要があります。workers を先にインポートすると PyAutoGUI が読み込まれ、Qt が Windows DPI awareness を設定できなくなる可能性があります。

## 検証チェック

正式なテストスイート、lint、CI 設定はありません。変更後は該当する非ゲームチェックを手動で実行してください：

- Python コンパイル/インポートチェック
- worker レジストリとテンプレート検証
- 更新 ZIP/展開チェック
- ロック/アップデータチェック
- AMD64 PE 検証
- パッケージ化後のスモークスタート
